Help & how it works
Everything you need to know to use Inside Rank — start here if it's your first time.
🎯 What is Inside Rank?
Inside Rank is a ranking intelligence dashboard built specifically for the InsideCasino affiliate network. It reads data exported directly from Google Search Console (GSC) and lets you compare two time periods side by side — so you can see exactly which keywords improved, declined, appeared, or disappeared.

It covers three sites: insidecasino.ca 🇨🇦, insidecasino.co.nz 🇳🇿, and insidecasino.com 🇬🇧. Each site has its own set of snapshots — switching sites (top-left sidebar) switches everything.
ⓘ Why not just use Ahrefs? Ahrefs only crawls the top ~30 SERP results, so anything ranking 31–100 shows as "Lost" even when it's still there. GSC reports actual impressions for every position, giving you the full picture.
🔗 GSC Live — setup guide & FAQ

GSC Live connects Inside Rank directly to your Google Search Console so you can pull ranking data with one click — no CSV exports needed. The setup is a one-time process that takes about 10–15 minutes.

📌 Step-by-step setup
Step 1 — Go to Google Cloud Console
Open console.cloud.google.com in a new tab. Sign in with the Google account that has access to Google Search Console. If you're not sure which account that is, check by going to search.google.com/search-console first.
Step 2 — Create a new project
At the very top of the page, click the project name dropdown (it might say "Select a project" or show an existing project name). Click New Project, give it a name like Inside Rank, and click Create. Google will redirect you to the new project's dashboard.
Step 3 — Enable the Search Console API
In the left sidebar click APIs & Services, then Library. In the search bar type Google Search Console API and click the result. Click the blue Enable button. This tells Google your project is allowed to read Search Console data.
Step 4 — Set up the OAuth consent screen
In the left sidebar click APIs & Services → OAuth consent screen. Select External and click Create. You'll see a form — fill in an App name (anything, e.g. Inside Rank) and your email address for "User support email" and "Developer contact information". Click Save and Continue three times until you reach the Summary page, then click Back to Dashboard.
Step 5 — Add test users
Still on the OAuth consent screen page, scroll down to the Test users section. Click + Add Users and add the email address of everyone who will use Inside Rank (including yourself). Click Save. Important: anyone not added here will get an "Access blocked" error when they try to connect.
Step 6 — Create a Client ID
Go to APIs & Services → Credentials. Click + Create Credentials at the top of the page and select OAuth client ID. Under "Application type" choose Web application. Scroll down to Authorised JavaScript origins, click + Add URI and enter exactly: https://insiderank.netlify.app. Click Create.
Step 7 — Copy and save the Client ID
A popup appears showing your new Client ID — a long string ending in .apps.googleusercontent.com. Copy it. Then go to Settings → GSC API in Inside Rank, paste it in the box and click Save Client ID.
Step 8 — Connect and pull data
Go to GSC Live. Click Connect Google Search Console. A Google sign-in popup appears — choose your account and click Allow. Once connected, choose your website from the property dropdown, pick a date range, and click Pull data.
❓ Common questions
  • "Access blocked: app not verified" — This means your email wasn't added as a test user. Go back to Google Cloud → APIs & Services → OAuth consent screen → Test users → add your email → Save. Then try connecting again.
  • "The popup is blocked" — Your browser is blocking the Google sign-in popup. Look for a blocked popup icon in the address bar, click it, and choose "Always allow popups from this site".
  • The connection expires after 1 hour. This is a Google security limit. Click Connect again — if you're still logged into Google in the same browser, it reconnects instantly with no popup.
  • I don't see my GSC property in the dropdown. Make sure you're connecting with the same Google account that has access to that property in Search Console. You can check at search.google.com/search-console.
  • What does "Content Gap" mean? It shows search queries where Google is showing your website in results and getting impressions — but you don't have a dedicated page built for that topic yet. These are content opportunities. Sort by Impressions to find the biggest ones.
  • Adding a new team member. Go to Google Cloud → OAuth consent screen → Test users → add their email. They then go to GSC Live on their own device and click Connect with their own Google account.
  • How much data does one pull get? Up to 25,000 queries per pull. For most websites this covers everything. If you have more, the top 25,000 by impressions are returned.
① Step 1 — Pull your data from GSC Live
Inside Rank pulls data directly from Google Search Console — no CSV exports or file uploads needed.

How to pull data:
  • Go to GSC Live in the left sidebar
  • If it's your first time, follow the one-time setup instructions to connect your Google account (takes about 10–15 minutes)
  • Once connected, select your website from the GSC Property dropdown
  • Choose a date range — Last 3 months, Last 6 months, or a custom range
  • Click Pull data — Inside Rank fetches up to 25,000 queries directly from Google
Each pull creates a snapshot — a saved record of all your keyword rankings for that date range. Pull different date ranges to build up a history you can compare.

Setting Period A and Period B: In the Pull Data tab, every snapshot has a Set A and Set B button. Period A = the older/baseline period. Period B = the current/newer period. The comparison always runs A → B. You can see and change the active periods in the top bar at any time.
⚠ Each site is separate. Data pulled while insidecasino.ca is selected only appears under .ca. Switch sites using the sidebar to manage different properties separately.
③ Step 3 — Sharing data with your team (Supabase sync)
By default, snapshots only exist in your own browser. Inside Rank uses a shared Supabase database to sync snapshots across the team automatically.

For Joy (admin) — one-time setup:
  • Go to Settings → Team sync — you'll see the full admin panel
  • Run the SQL shown there in your Supabase SQL editor to create the required tables (there are now two: one for GSC snapshots, one for GA4 data)
  • After pulling GSC data, click ↑ Sync next to each snapshot in the GSC Live → Pull Data tab to push it to the team
  • After uploading GA4 CSVs in Page Performance, they are pushed to the team automatically — no extra step needed
For all other team members — nothing to set up:
  • Log in with your email and the team password
  • GSC snapshots and GA4 data both appear automatically — check the Page Performance tab to see the GA4 data loaded
  • Cards showing a 🌐 Team badge were synced by the admin
  • Use Settings → Refresh GA4 data to force a manual pull if data seems stale
🔒 Your GSC data stays in your own Supabase project. Inside Rank never stores it externally.
📋 What each tab does
  • Overview — The dashboard. Shows KPI cards (total clicks, impressions, avg position, top 3, pos 11–30, and the Ahrefs blind spot pos 31–100), a position distribution chart comparing A vs B, health summary (improved/declined/new/lost/stable counts), and top 10 winners and losers.
  • Keywords — Full keyword-level table. Every keyword GSC has tracked, with its position in both periods, delta, clicks, impressions, CTR, status badge, and the URL it's assigned to. Filterable by status, position range, and search. Sortable by any column.
  • Pages — Two sub-views (toggle at top):
      — Expanded view: Pages grouped by category (Casino Reviews, Slots, Payment Methods, etc.). Click any row to expand it and see all keywords for that page.
      — Ahrefs-style table: Flat table showing URL, Page type, Status, Traffic, Change %, Keywords, KW change, Top keywords (primary + 2 secondary inline, with a "+N more" dropdown), and estimated volume.
  • SERP Tracker — Pulls live Google rankings for every keyword in Period B (not just top 30). Shows your real position right now, Drift (how much your live position differs from the GSC average), GSC Δ (movement A→B), SERP features (Snippet, PAA, Local, Video), and bojoko.ca / bonus.ca positions side by side. Rows where a competitor is beating you are highlighted red. Filterable by position range, SERP feature, competitor presence, GSC movement, and position mismatch. Exportable to CSV.
  • Page Performance — Combines GA4 engagement data with GSC ranking data into one page-level table. Shows Visitors, Views, Time on page (all with period-over-period change columns), and the ranking keywords for each page pulled from your GSC snapshot. Requires uploading GA4 CSV exports separately — see the full guide further down this Help page.
  • URL Tracker — A focused before/after comparison tool for pages you've actively worked on. Add specific URL paths you've made changes to (content rewrites, new FAQ sections, internal linking updates, etc.) and instantly see how Clicks, Impressions, and Average Position changed between Period A and Period B. Also shows a full keyword-level breakdown per page. See the complete guide further down this Help page.
  • GSC Live — Pull data directly from Google Search Console. Manage snapshots and set which is Period A and which is Period B.
  • Settings — Configure team sync (Supabase), Serper.dev API key, and active site.
📈 Understanding the numbers
Position badges — colour-coded by rank bucket:
1.0Positions 1–3 (green) 8.5Positions 4–10 (amber) 18.2Positions 11–30 (blue) 45.0Positions 31–100 (grey)
Status badges — how a keyword moved between Period A and Period B:
NewRanking in B but not in A ImprovedPosition moved up by ≥0.5 DeclinedPosition dropped by ≥0.5 StableLess than 0.5 position change LostWas ranking in A, not in B
Δ POS (delta position) — a ↓ number means the position number went down (ranking improved). A ↑ number means the position number went up (ranking declined). Lower position number = higher on Google.

Pos 31–100 KPI card — this is the "Ahrefs blind spot." These are keywords that are still ranking and getting impressions but Ahrefs won't show them because it only indexes the top 30 results. This is why GSC is used as the source of truth.

Chart.csv ✓ vs ⚠ — When you import a full ZIP, it includes a Chart.csv with aggregate totals that exactly match what GSC shows in its UI. If a snapshot shows ✓ Chart.csv the click/impression totals are exact. If it shows ⚠ No Chart.csv the totals are summed from the top-1,000 keywords only and may be slightly lower than the GSC UI.

URL matching — Keywords are matched to URLs in two ways: Exact means the keyword and URL came from the same combined CSV row. Inferred means the tool matched the keyword to a URL by comparing tokens in the keyword against tokens in the URL path (e.g. "interac casino" → /payment-methods/interac/). Inferred matches are good but not guaranteed.
🔍 Using the SERP Tracker
The SERP Tracker fetches live Google rankings for every keyword in your Period B snapshot — not just the top 30 — and compares them against bojoko.ca and bonus.ca in real time.
  • Make sure Period B is set to your most recent snapshot
  • Click Run SERP check — results are fetched in order of impressions (highest traffic keywords first)
  • You can click Cancel to stop mid-run and work with what's been fetched so far
  • Results are saved to your browser and load automatically next session
  • Use Export CSV to download the full dataset for reporting
Columns explained:
  • GSC Pos (B) — the average position from your Period B snapshot
  • Live Pos — the actual Google position fetched right now
  • Drift — how far the live position differs from the GSC average. A large drift means the ranking is volatile (GSC averages it out over the whole period, so a keyword that moved recently will show drift)
  • GSC Δ — how the GSC position changed between Period A and Period B
  • SERP Features — ✦ Featured Snippet, 💬 People Also Ask, 📍 Local Pack, ▶ Video
  • bojoko.ca / bonus.ca — their live position for that keyword. ▲ ahead = they outrank you. ▼ behind = you outrank them. Rows where a competitor is ahead are highlighted in red.
Filters:
  • Position range — narrow to top 3, top 10, 11–20, etc.
  • SERP features — find all keywords where a Featured Snippet or PAA box is present
  • Competitor presence — "Outranking us" is the most useful: shows only keywords where bojoko or bonus is ahead of you right now
  • Position mismatch — toggle to show only keywords where live position and GSC average differ by 5+ positions (volatile rankings worth monitoring)
⏱ A full SERP check takes roughly 0.35 seconds per keyword to avoid rate limits. 500 keywords ≈ 3 minutes. The progress bar shows an estimated time remaining.
❓ Common questions
  • I imported a file but see no data. Make sure you're on the right site (check the sidebar — .ca, .co.nz, or .com). Also check that you have Period A and Period B both set in the top bar. If only one snapshot exists, set it as Period B and the Overview will show current data without a comparison.
  • My keyword shows "Lost" but I know it's still ranking. This usually means it fell below position 30 in Period B but is still ranking at 31–100. Check the Pos 31–100 KPI card on the Overview. You can also filter the Keywords tab by "31–100 (Ahrefs blind)" to see them all.
  • The URL column is blank for some keywords. GSC Queries exports don't include URL data by default. Import a Pages CSV alongside your Queries CSV (or use the full ZIP export) — the tool will cross-reference and fill in URLs automatically.
  • Can I compare more than two periods? Yes — import as many snapshots as you want, then use the Period A and Period B pickers in the top bar to choose any two.
  • Team members can't see my snapshots. Make sure you clicked ↑ Sync next to the snapshot in the GSC Live → Pull Data tab. Team members connect automatically — they don't need to enter any credentials. If they still can't see it, ask them to click Refresh team snapshots in Settings.
  • The Overview shows 0 clicks/impressions. This happens when you import a single Queries CSV without a Chart.csv. The tool totals from the top-1,000 rows — if your site has very low traffic it may still show correctly, but for large sites use the full ZIP export.
  • Position numbers look like averages, not exact ranks. That's correct — GSC reports the average position across all times a keyword appeared in results during the period. A keyword at "4.3" means it appeared sometimes at position 3, sometimes at position 5, averaging 4.3.
📊 Page Performance — complete guide
The Page Performance tab combines Google Analytics 4 (GA4) engagement data with your GSC ranking data to give you a single view of every page: how many people visited, how long they stayed, and which keywords are driving them there — all with period-over-period comparisons.
📊 What data sources does it use?
The tab joins two completely separate data sources on the page URL:
  • Google Analytics 4 (GA4) — provides Visitors (sessions), Views (pageviews), and Time on page. You export this yourself as CSVs from the GA4 interface.
  • GSC snapshots — provides Ranking keywords, average position, and position change. These are the same snapshots pulled via GSC Live — no extra work needed for this part.
ⓘ The tool matches pages between GA4 and GSC by their URL path (e.g. /casinos/reviews/winspirit/). If a page appears in GA4 but has no GSC keywords, the Ranking keywords column will be blank for that row — this is normal for pages that don't rank organically.

Minimum to get started: You only need Period B — Landing pages to see data. Everything else adds comparison columns. Upload just that one file and the table will populate with current Visitors, Time on page, and Ranking keywords immediately.
💡 Match your date ranges to your GSC snapshots. For example, if your GSC Period B snapshot covers Feb 19 – May 18, set the same date range when downloading your GA4 CSVs. This ensures the GA4 engagement data and GSC ranking data reflect the same time window.
📺 Step-by-step: exporting from GA4
Landing pages report (for Visitors and Time on page):
  1. Go to analytics.google.com and select your property (Inside Casino Canada GA4)
  2. In the left menu click Reports → Engagement → Landing pages
  3. In the top right, click the date range (e.g. "Feb 19 – May 18, 2026") and set it to your desired period
  4. Make sure the segment filter at the top shows Organic traffic — if not, add it by clicking the + Add comparison button and selecting Organic traffic. This ensures the numbers match your GSC data.
  5. Click the download icon (looks like ↓ with a line) in the top right of the report → select Download CSV
  6. The file will be named something like Landing_page_Feb_19__2026_to_May_18__2026.csv
  7. Repeat for the other date range (Period A)
Pages and screens report (for Views):
  1. Same steps as above but at step 2, click Pages and screens instead of Landing pages
  2. Set the same date range and Organic traffic segment
  3. Download CSV — the file will include a Views column
  4. Repeat for Period A if you want the Views change % column
⚠ Important: GA4 exports only the first 5,000 rows by default. If your site has more than 5,000 landing pages, not all pages will be included. For most affiliate sites this won't be an issue.
📈 Understanding the columns
  • Page — the URL path of the landing page (e.g. /casinos/reviews/winspirit/) and its inferred category (Casino Reviews, Slots, Payment Methods, etc.). Clicking the URL opens the live page.
  • Visitors — total GA4 sessions for that landing page in Period B. A "session" is one visit by one user — if the same person visits twice they count as 2. The grey "prev: X" underneath shows the Period A value for easy reference.
  • Visitors Δ — the percentage change in visitors from Period A to Period B. Green = more visitors. Red = fewer visitors. Only shows when Period A Landing pages is uploaded.
  • Views — total pageviews for that page in Period B. Unlike Visitors, if one person views the same page 3 times in a session that counts as 3 views. Comes from the Pages and screens report. Shows "—" until Period B Pages and screens is uploaded.
  • Views Δ — percentage change in views from Period A to B. Only shows when both Period A and Period B Pages and screens files are uploaded.
  • Time on page — GA4's "Average engagement time per session" — how long visitors typically spend on that page. Formatted as minutes and seconds (e.g. 1m 30s). Longer time generally means better engagement. The grey "prev:" value shows the Period A figure.
  • Ranking keywords — pulled from your GSC Period B snapshot, matched to the page URL. Shows the top 2 ranking keywords with their current position badge and position change arrow (↓ improved, ↑ declined). Click the "+N more" button to see all ranking keywords for that page in a dropdown with impressions data.

🎯 KPI cards at the top
The four summary cards above the table show totals and averages across all pages in your filtered view:
  • Total visitors — sum of all sessions across all pages. The ↑/↓ delta and percentage show the change vs Period A when that file is uploaded.
  • Total views — sum of all pageviews. Delta only appears when both Period A and Period B Pages and screens files are uploaded.
  • Avg time on page — average engagement time across all pages. Delta shows when Period A Landing pages is uploaded.
  • Pages with GSC data — how many of your GA4 pages were successfully matched to GSC keyword data. If this number is much lower than the total, it usually means your GA4 URLs have trailing slashes and your GSC URLs don't (or vice versa) — the tool handles this automatically in most cases.
🔍 Filters and sorting
  • Search — filters by URL path or keyword name. Type /casinos to see only casino pages, or type a keyword like interac to find pages ranking for that term.
  • All page types — filters by category inferred from the URL structure (Casino Reviews, Slots, Payment Methods, Software, Bonuses, Ontario Casinos, etc.).
  • Sort — sort by Visitors, Visitor change, Views, Time on page, or number of Ranking keywords. Default is Visitors (highest first).
  • All pages / Has ranking keywords / Has top-10 keyword / No ranking data — filter to focus on pages with strong GSC presence, pages in the top 10, or pages with no organic keyword data at all (useful for identifying pages that get traffic from other sources).
Tip: Sort by Visitor change (descending) to see your biggest-growing pages. Sort by Visitors with the Has top-10 keyword filter to see your highest-traffic, best-ranking pages — those are your most valuable assets to protect and grow.
❓ Page Performance — common questions
  • Ranking keywords column shows "No GSC match" for most pages. This means the tool couldn't match the GA4 URL to a GSC URL. The most common cause: GA4 uses trailing slashes (/casinos/) and GSC doesn't, or vice versa. The tool tries both variants automatically — if it's still not matching, check that your GSC Period B snapshot is set correctly in the top bar.
  • Views column shows "—". You need to upload the Pages and screens CSV (Period B). That's the only GA4 report that includes the Views column — the Landing pages report doesn't have it by default.
  • Views Δ column shows "—". You also need the Pages and screens CSV for Period A (the previous period card in the top-right of the upload grid). Upload that file and the change column will populate.
  • The numbers don't match what I see in GA4. Make sure you're exporting with the Organic traffic segment applied. Without it, GA4 includes all traffic sources (direct, paid, social, email) and the numbers will be much higher than what GSC shows. Also double-check your date ranges match your GSC snapshot periods exactly.
  • Some pages in GA4 don't appear in the table. The table is built from the Period B Landing pages file — pages that appear in your GA4 Pages and screens report but not in Landing pages won't show up. Use the Landing pages report (not Pages and screens) as your primary upload for Period B.
  • Time on page shows 0s for some pages. This is normal for pages where GA4 couldn't measure engagement time — usually pages where the visitor left without any interaction, or single-page sessions where GA4 couldn't calculate duration. It doesn't necessarily mean the page performed poorly.
  • Can I export this data? Yes — click the Export CSV button in the top right of the Page Performance tab. The export includes all columns: URL, category, visitors (current and prev), views (current and prev), time on page (current and prev), ranking keyword count, top keyword, and its position.
🎯 URL Tracker — complete guide
The URL Tracker is a focused before/after tool built specifically for tracking the impact of changes you've made to individual pages. Instead of looking at hundreds of pages at once, you add only the pages you've worked on and get a clean side-by-side comparison showing exactly what moved — and by how much — between your two GSC periods.

When to use it: Any time you've made a deliberate change to a page and want to know if it worked. Examples:
  • Rewrote the intro or meta title of a casino review
  • Added an FAQ section to a payment methods page
  • Updated internal linking to a key landing page
  • Published a new slot review and want to track early ranking movement
  • Made technical changes (schema markup, page speed, canonical) and want to see impact
📋 Before you start — set your periods
The URL Tracker pulls all its data from the GSC snapshots you've already imported. It doesn't need any new files — just make sure you have the right periods set.

What Period A and Period B mean for tracking changes:
  • Period A = the period before you made your changes. This is your baseline.
  • Period B = the period after your changes. This is your "current" state.
The comparison always runs A → B, so green deltas mean things improved in Period B compared to Period A.

How to check which periods are set: Look at the top bar in Inside Rank. The two date pickers show which snapshots are Period A and Period B. If they're not what you want, click either picker and select a different snapshot.
💡 Tip: For the most meaningful comparison, choose periods of equal length — e.g. both 3 months. A 3-month vs 1-week comparison will almost always show lower numbers in the shorter period, not because rankings declined but because there were fewer days of data.
① Adding pages to track
Step 1: Click URL Tracker in the left sidebar.

Step 2: Click the Add URL button in the top right of the tracked pages card.

Step 3: A modal window will appear with a text box. Enter one URL path per line. Use the path only — no domain name, no https.

📈 Understanding the comparison table
The summary table shows one row per tracked URL with the following columns:
  • Page — the URL path and your note (if you added one).
  • Clicks A — total organic clicks in Period A (your baseline). Sourced from your GSC snapshot.
  • Clicks B — total organic clicks in Period B (after your changes). Shown in bold.
  • Clicks Δ — the absolute change. +12 means 12 more clicks in Period B. -5 means 5 fewer.
  • Impressions A / B / Δ — same as clicks but for impressions (how many times your page appeared in search results). A rising impression count with flat clicks often means you're appearing for more queries but not yet converting them — watch position alongside this.
  • Avg Pos A — your average GSC position across all keywords for that page in Period A.
  • Avg Pos B — average position in Period B, shown as a colour-coded badge (green = top 3, amber = top 10, blue = 11–30, grey = 31+).
  • Pos Δ — the change in average position. ↓ 3.2 means the position number dropped by 3.2 — which is good (closer to position 1). ↑ 1.8 means position increased (moved further from position 1 — worse).
  • Note — the note you entered when adding the URL, if any.
ⓘ No data showing? If Clicks A, Impressions A, and Avg Pos A all show "—" it means that page had no ranking keywords in your Period A snapshot. This could mean the page was brand new, didn't rank for any tracked queries in that period, or the URL in the tracker doesn't exactly match the URL in your GSC data. See the troubleshooting section below.
🔍 Keyword breakdown cards
Below the summary table, each tracked page gets its own keyword breakdown card. This shows every individual keyword that page ranked for in either Period A or Period B, giving you a granular view of exactly which queries moved.

Columns in the keyword breakdown:
  • Keyword — the search query.
  • Pos A — average position in Period A (shown in grey — your baseline).
  • Pos B — average position in Period B (colour-coded badge — your current state).
  • Pos Δ — ↓ 4.1 = improved by 4.1 positions. ↑ 2.3 = declined by 2.3 positions. ≈ = essentially flat.
  • Impr. A / Impr. B — impressions for that specific keyword in each period. Rising impressions on a keyword usually means it's moving up and appearing for more search variations.
Summary badges on each card:
↑ 5 improved5 keywords moved to a better position
↓ 2 declined2 keywords moved to a worse position
+3 new3 keywords appeared in Period B that weren't in Period A
1 lost1 keyword ranked in Period A but not in Period B
Keywords are sorted by impressions (highest first) so the most impactful ones appear at the top.
💡 How to interpret results — practical examples
Scenario 1 — Clear improvement:
Clicks A: 12 → Clicks B: 31 +19  ·  Avg Pos A: 14.2 → Pos B: 8.9 ↓ 5.3
Your changes worked. Position improved by 5.3 places and clicks nearly tripled. Look at the keyword breakdown to see which specific queries drove the improvement — those are the ones your changes resonated with.

Scenario 2 — Impressions up, clicks flat:
Clicks A: 8 → Clicks B: 9 +1  ·  Impr A: 420 → Impr B: 890 +470  ·  Avg Pos A: 22.1 → Pos B: 18.4 ↓ 3.7
You're moving up and appearing for more queries, but CTR is low because you're still in positions 11–30. This is a promising sign — keep optimising. Focus on meta title and description to improve click-through rate.

Scenario 3 — Position improved but clicks dropped:
Clicks A: 45 → Clicks B: 28 -17  ·  Avg Pos A: 8.4 → Pos B: 6.1 ↓ 2.3
Position improved but clicks fell. Check if Period B had fewer days or if it covered a seasonal low. Also look at the keyword breakdown — you may have improved for low-volume queries while losing ground on the high-volume ones.
❓ URL Tracker — common questions
  • The table shows "—" for all Period A columns. This means the page had no GSC keyword data in Period A. Most likely causes: the page was created after Period A ended, it didn't rank for any queries in that period (new or thin content), or the URL in the tracker doesn't match the URL format in your GSC snapshot. Try checking the Keywords tab and filtering by that URL to confirm how it appears in your GSC data.
  • All columns show "No data" in Period B. The page has no ranking keywords in Period B either. Check that the URL path is correct (exact match including trailing slash). Also confirm your Period B snapshot is set — look at the comparison label above the table which shows which snapshots are being used.
  • The URL I added doesn't match. GSC sometimes stores URLs with trailing slashes (/casinos/reviews/winspirit/) and sometimes without. The tracker tries both variants automatically. If still no match, go to the Keywords tab, search for a keyword you know ranks for that page, and check exactly how the URL appears in the URL column — then add that exact format to the tracker.
  • Position improved but I want to see it at the keyword level. The "Avg Pos" column in the summary table averages across all keywords for that page. A page ranking for 50 keywords will show an average that may mask big moves on individual terms. Scroll down to the keyword breakdown card for that page — sort mentally by Impr. B to focus on the high-traffic keywords first.
  • The comparison period labels say "No Period A". You only have one snapshot imported. Import a second GSC ZIP or CSV for the previous period, then set it as Period A using the top bar picker. You need two snapshots for the comparison to work.
  • Can I track the same URLs across multiple sites? Yes — URLs are stored per site. Switch to .co.nz or .com in the sidebar and add URLs there independently. Each site has its own separate tracker list.
  • I made changes mid-period — will that affect the data? Yes, and it's worth being aware of. GSC reports average position over the entire period. If you made changes on day 15 of a 30-day period, Period B only captures 15 days of post-change data averaged in with 15 days of pre-change data. For cleanest results, export a new GSC snapshot that starts after your change date and use that as Period B.
  • Can I share my tracked URL list with the team? Currently tracked URLs are stored in your own browser. They don't sync via Supabase. Each team member manages their own list. If you want others to track the same pages, share the URL list with them and they can add them manually.
  • Can I export the data? Yes — click Export CSV in the top right. The export includes one row per URL with all A/B values and deltas for clicks, impressions, and position, plus your notes.
